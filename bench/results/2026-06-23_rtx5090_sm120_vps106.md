## 7. Conclusion

This second RTX 5090 validates the same production picture:

```text
v1.2.15 generic fallback is healthy on sm_120:
  POSTCHECK ok=1,
  live pool accepted,
  rejected=0,
  submit_timeout=0,
  run.sh auto-restart wrapper is active,
  GPU utilization reaches 100%.
```

Performance:

```text
controlled kernel total:  ~319.5 TH/s
live 60s after 22m:       ~277.9 TH/s
15m window after 22m:     ~286.4 TH/s
job_abort/pure-ish live:  ~302.7 TH/s
```

Main live gap drivers observed again:

```text
short found attempts,
CPU proof path around 96ms,
submit_wait around 0.67s,
job aborts from new pool jobs,
occasional stale share when a job changes during/near submit.
```

Next step remains unchanged:

```text
Keep generic and sm86-g8 stable.
Build a CUDA 13 native ARCH=sm_120 candidate package through workflow/build-worker.
Validate it against this generic fallback baseline before promoting any sm_120
package to automatic install selection.
```
