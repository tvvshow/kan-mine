Code-level follow-up confirmed the interpretation:

```text
perf attempt elapsed covers mine_plain_proof() only;
live hashrate windows sample g_live_draw_count over wall-clock;
the pool submit path waits synchronously for submit_resp;
therefore submit_wait periods reduce live table TH/s even though they are not
included in per-attempt perf elapsed.
```

## 5. Baseline conclusion

Current status:

```text
v1.2.15 generic fallback is healthy on RTX 5090 / sm_120:
  correctness smoke passed in the session,
  pool accepted shares,
  rejected=0,
  submit_timeout=0,
  live 60s around 300 TH/s.
```

Production profile status:

```text
This is not a tuned sm_120 profile.
It is a generic compatibility fallback baseline.
```

Next production-grade improvement path:

```text
1. Keep existing generic and sm86-g8 release profiles unchanged.
2. Build a CUDA 13 native ARCH=sm_120 candidate package in the release workflow
   or another controlled build worker.
3. Validate POSTCHECK, controlled benchmark, and live pool accepted data.
4. Only after documented validation should install_kan.sh select an sm120 tuned
   package automatically.
5. Separately measure found/proof/submit wall-clock overhead; async submit/proof
   must be an opt-in production feature until fully validated.
```
